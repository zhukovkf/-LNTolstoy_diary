# Автопостинг дневника Толстого в Telegram (@LNTolstoy_diary)

## Что в этой папке

- `post_next_entry.py` — публикует одну следующую запись из дневника, помнит,
  на чём остановился.
- `tolstoy_diary_autopost_modern.csv` — записи (2670 шт., по одной на пост).
- `.github/workflows/post.yml` — расписание GitHub Actions: каждый день в
  09:00 по Москве.
- `state.json` — появится сам после первого запуска, хранит номер следующей
  записи. Важно, чтобы он сохранялся обратно в репозиторий (workflow это
  делает автоматически) — иначе каждый день будет уходить одна и та же запись.

## Настройка (один раз, минут 5)

1. Создать репозиторий на GitHub (можно приватный) и загрузить туда всё
   содержимое этой папки как есть, с сохранением структуры (включая скрытую
   папку `.github`).
2. В репозитории: **Settings → Secrets and variables → Actions → New
   repository secret**.
   Имя: `TELEGRAM_BOT_TOKEN`, значение — токен вашего бота от @BotFather.
3. Проверить, что бот действительно администратор канала @LNTolstoy_diary
   с правом публиковать сообщения (это уже сделано, судя по вашим словам).
4. Готово. Публикация пойдёт по расписанию сама. Проверить прямо сейчас, не
   дожидаясь 9 утра: вкладка **Actions** → workflow «Daily Tolstoy post» →
   **Run workflow**.

## Формат поста

```
📖 <дата по дневнику>

<текст записи>
```

Записи длиннее лимита Telegram (4096 символов) автоматически разбиваются на
несколько сообщений подряд.

## Если что-то не публикуется

Смотреть вкладку **Actions** в репозитории — там лог каждого запуска и
понятная ошибка, если токен неверный или канал недоступен боту.
